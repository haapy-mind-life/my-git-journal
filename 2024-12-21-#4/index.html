    아래는 GitHub Actions를 사용해 MkDocs 사이트를 자동 빌드 및 배포(gh-deploy) 하는 방법에 대한, 완전 초보자용 가이드입니다.

    이 가이드는 이미 다음과 같은 상태를 가정합니다:

    1. 프로젝트 구조

    requirements.txt, app.py (Streamlit 코드), mkdocs.yml(또는 생성 예정), docs/ 폴더 등이 포함된 레포지토리

    app.py에는 프로젝트에서 사용 중인 mkdocs 빌드 로직이 포함됨(실제로 빌드/배포는 로컬이나 GitHub Actions에서 실행)

    1. GitHub 리포지토리

    이 프로젝트가 호스팅될 GitHub 저장소가 있습니다(예: https://github.com/<사용자명>/<레포명>.git).

    로컬 PC에서 Git으로 관리 중이거나, GitHub Codespaces/VSC를 사용해 관리 중

    1. main 브랜치

    GitHub에서 기본 브랜치를 main으로 사용한다고 가정(만약 master가 기본이라면 아래 모든 main을 master로 바꿔주세요).

    1. GitHub Pages를 활용한 MkDocs 호스팅 개념

    MkDocs: Markdown 문서를 정적 사이트 형태로 빌드해주는 툴

    GitHub Pages: GitHub에서 제공하는 정적 호스팅(HTML/CSS/JS 파일 등을 배포 가능)

    mkdocs gh-deploy: MkDocs가 로컬 Git 리포지토리에서 gh-pages 브랜치로 정적 파일을 푸시해주는 명령어

    문제는, Streamlit Cloud 환경에서는 .git 폴더나 원격 저장소 설정(origin)이 없어서 mkdocs gh-deploy가 실패합니다. 이를 GitHub Actions가 대신 해주면, main 브랜치에 변경이 들어올 때마다 자동으로 MkDocs를 빌드하고 gh-pages로 배포해주므로 편리합니다.

    1. GitHub Actions 설정 개요

    GitHub Actions는 GitHub 레포지토리 안에 .github/workflows 폴더를 만들어서, 그 안에 YAML 파일을 작성하면 동작합니다. 이 YAML 파일을 “워크플로우”라고 부르며, main 브랜치로 push가 발생할 때마다 자동으로 실행됩니다.

    2.1 폴더 & 파일 구조 예시

    my-git-journal/ ├─ .github/ │ └─ workflows/ │ └─ mkdocs-deploy.yml <-- 여기서 자동 배포 설정 ├─ docs/ │ └─ index.md ├─ mkdocs.yml ├─ app.py ├─ requirements.txt └─ ...

    1. GitHub Actions 워크플로우 파일 작성 (mkdocs-deploy.yml)

    2. 프로젝트 루트 디렉터리에 .github/workflows 폴더 생성(없으면 새로 만듦)

    3. 그 안에 mkdocs-deploy.yml 라는 이름으로 파일 작성

    4. 내용 예시:

    name: Deploy MkDocs to GitHub Pages

    on: push: branches: [ "main" ] # main 브랜치에 push될 때 동작

    jobs: build-deploy: runs-on: ubuntu-latest steps: # (1) 소스코드 가져오기 - name: Check out the repository uses: actions/checkout@v3

      # (2) Python 설치 (3.9 버전 예시)
  - name: Set up Python
    uses: actions/setup-python@v2
    with:
      python-version: "3.9"

  # (3) 의존성 설치
  - name: Install dependencies
    run: |
      pip install --upgrade pip
      pip install -r requirements.txt
      # requirements.txt에 mkdocs, mkdocs-material, streamlit 등 이미 작성됨

  # (4) mkdocs build & gh-deploy
  - name: Build and deploy to GitHub Pages
    run: |
      mkdocs build
      mkdocs gh-deploy --force

    위 스크립트가 하는 일은:

    1. main 브랜치로 push될 때 자동 트리거

    2. Ubuntu 환경에서 Python 3.9 설치

    3. pip install -r requirements.txt (여기에 mkdocs, mkdocs-material, 등등이 들어있어야 함)

    4. mkdocs build → mkdocs gh-deploy --force

    1. main 브랜치로 커밋 & 푸시 → GitHub Actions 동작

    2. 로컬에서 .github/workflows/mkdocs-deploy.yml 파일 생성/수정 후 git add . && git commit -m "Add mkdocs actions"

    3. git push origin main

    4. GitHub에 가서 Actions 탭을 보면, “Deploy MkDocs to GitHub Pages” 워크플로우가 실행되는 것을 확인 가능

    5. 빌드가 성공하면, gh-pages 브랜치가 자동 생성되어 정적 파일들이 올라갑니다.

    1. GitHub Pages 설정

    2. GitHub 저장소 → Settings → Pages

    3. “Branch” 항목에서 “gh-pages”를 선택 후 “Save”

    4. 잠시 기다리면, 아래쪽에 “Your site is published at ~~” 라는 문구와 함께 링크가 표시

    5. 브라우저에서 해당 링크(https://사용자명.github.io/레포명/)로 접속하면 MkDocs 정적 사이트가 보여야 합니다.

    1. 사용 시나리오 정리

    2. 문서 작성 & 수정

    docs/ 폴더에 .md 파일 작성 (Front Matter, 내용 등)

    git add . && git commit -m "Update docs" → git push origin main

    1. GitHub Actions 자동 빌드

    Actions가 pip install -r requirements.txt → mkdocs build → mkdocs gh-deploy --force

    gh-pages 브랜치에 정적 파일 푸시

    1. GitHub Pages 자동 업데이트

    사용자는 “https://사용자명.github.io/레포명/”로 접속하면 항상 최신 문서 확인 가능

    1. 유의사항 & 자주 묻는 질문

    2. Q. “Actions에서 권한 문제 발생!”

    보통 퍼블릭 레포는 문제 없지만, 프라이빗 레포에서 Pages를 쓰려면 GitHub 유료 요금제, 그리고 Actions 설정에서 “Allow GitHub Actions to create and approve pull requests” 등 권한 조정이 필요할 수 있음

    또는 “Page build and deployment” 권한이 필요

    1. Q. “mkdocs gh-deploy --force” 써야 하나요?

    --force는 “이미 gh-pages 브랜치가 있어도 강제 덮어쓰기” 옵션입니다. 보통 충돌 방지 용도로 자주 사용

    1. Q. 로컬에서 mkdocs gh-deploy 실행하면 안 되나요?

    가능하지만, Streamlit Cloud 환경에선 .git이 없어서 실패합니다.

    GitHub Actions 자동화가 훨씬 편리

    1. Q. 사이트 주소가 복잡하거나, 커스텀 도메인 쓰고 싶어요.

    GitHub Pages의 “Custom domain” 기능 사용. “Settings → Pages → Custom domain”에 내 도메인을 연결하면 됩니다.

    1. 요약

    완전 초보자라면, GitHub Actions 설정 파일(.github/workflows/mkdocs-deploy.yml)만 추가하면 OK

    main 브랜치에 push → Actions가 MkDocs 빌드 & gh-deploy → GitHub Pages에 최신 문서 게시 → 사이트 자동 갱신

    응원의 말 "처음엔 복잡해 보이지만, 한 번 구축해두면 매번 수동으로 배포할 필요 없이 문서를 수정할 때마다 자동으로 GitHub Pages가 업데이트됩니다. 꾸준히 경험을 쌓으면 더욱 안정적이고 효율적인 개발 흐름을 만들 수 있을 거예요. 파이팅입니다!"
